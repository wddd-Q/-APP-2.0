## 一、设计原则

### 视觉风格: 水墨古风
- **主色调**: 米黄底 (#E8D5B7) + 深棕文字 (#2C1810)
- **强调色**: 朱砂红 (#C04040) — 用于警告/重要提示
- **辅助色**: 松石绿 (#4A7C59) — 用于正面/成功状态
- **金色**: (#C9A96E) — 用于灵石/声望等珍贵资源
- **背景**: 仿宣纸纹理 + 淡墨山水装饰

### 布局原则
- **左上→右下** 阅读顺序（中文自然流向）
- **固定顶栏** — 时间/资源/核心操作不随标签切换消失
- **标签导航** — 底部或左侧标签切换不同管理面板
- **浮层事件** — 随机事件以居中弹窗呈现，不打断主界面

## 三、关键交互

### 核心操作: 推进时间

```
[推进一月 ▶] 按钮 — 始终在顶栏右侧
快捷键: Space

点击后:
1. TimeManager.advance_month()
2. 所有订阅者依次处理（~0.2s完成）
3. UI 自动刷新（EventBus 信号驱动）
4. 若有事件触发 → 弹出事件对话框
```

### 核心操作: 招收弟子

```
[招收弟子] 按钮 — 在弟子面板中
点击后:
1. 弹出候选人列表（2-5人）
2. 每人显示: 姓名 | 年龄 | 灵根(可检测) | 粗略属性评估
3. 选择招收 → DiscipleController.recruit() → 刷新弟子列表
```

### 核心操作: 分配任务

```
弟子行右侧 [任务下拉] — 在弟子面板中
选项: 闭关修炼 | 炼丹 | 炼器 | 外出游历 | 镇守 | 教导 | 空闲
选择后 → DiscipleController.assign_task() → 刷新
```

### 核心操作: 建造设施

```
宗门总览 → 设施区 [建造] 按钮
点击后 → 弹出可选设施列表（显示成本和条件）
选择 → SectController.build_facility() → 刷新设施区
```

## 四、信息层次

| 层级 | 内容 | 刷新频率 |
|------|------|----------|
| **随时可见** | 时间、灵石、声望、弟子数 | 每次操作后 |
| **常驻面板** | 设施列表、弟子名册、资源库存 | 月度结算后 |
| **按需查看** | 弟子详情、丹药堂界面、世界地图 | 点击时加载 |
| **事件驱动** | 随机事件弹窗、通知徽标 | 触发时显示 |

## 五、无障碍考量

| 需求 | 实现 |
|------|------|
| 色盲支持 | 关键状态用图标+颜色双重表示 |
| 键盘导航 | Tab 切换焦点、Space 确认、Esc 关闭弹窗 |
| 字体缩放 | 最小字号 14px，支持 UI 缩放 |
| 屏幕朗读 | 关键 Label 设置有意义文本 |
| 操作确认 | 不可逆操作（解雇弟子/宣战）二次确认 |
