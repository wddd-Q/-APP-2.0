## 电梯演讲

> 一款以"掌门视角"为核心的修仙门派模拟经营游戏。你不是独行侠，而是一宗之主——从收徒、授业到资源经营和外交博弈，把破败山门建设成修真界的传说。

---

## 核心身份

### 一句话定位
修仙门派经营模拟，经典网文世界观 × RimWorld式管理深度。

### 核心卖点
1. **掌门视角**: 不再是单打独斗的修仙RPG，你管理整个宗门
2. **弟子即生命**: 每个弟子从灵根测试到飞升/坐化，有完整的人生故事线
3. **网文正统**: 严谨的九大境界体系，灵根/天劫/丹药/法器经典设定
4. **动态世界**: NPC宗门也在扩张衰落，不是你一个人在推动世界

## MDA 分析

### 美感 (Aesthetics)
| 类型 | 描述 |
|------|------|
| **叙事涌现** | 弟子间的恩怨纠葛、机缘奇遇、宗门兴衰，故事由系统生成 |
| **掌控与进步** | 从破败小派到修真巨擘的成长满足感 |
| **决策压力** | 有限的灵石与灵脉，无限的野心——取舍是常态 |
| **发现与探索** | 未知秘境、失传丹方、隐世高人、天才弟子 |

### 机制 (Mechanics)
- 回合制时间推进（每月/每年）
- 弟子属性养成：根骨、悟性、福缘、心性、魅力、资质
- 灵根系统：五行属性 + 品质判定
- 境界突破：九境修炼 + 天劫 + 丹药辅助
- 设施建设：修炼室/藏经阁/丹药堂/演武场/阵法殿/灵兽园
- 资源经济：灵石货币体系 + 灵脉开采 + 坊市交易
- 炼丹炼器：丹方收集 + 材料配比 + 成功率公式
- 外交系统：宗门声望 + 同盟/敌对关系 + 阵营选择

### 动态 (Dynamics)
- **培养循环**: 招收弟子 → 分配任务 → 资源积累 → 设施升级 → 更强弟子
- **资源压力**: 灵石总是不够用，弟子突破要丹药，设施升级要材料
- **风险回报**: 派弟子探秘境可能得奇遇也可能陨落
- **宗门竞争**: 其他宗门也在发展，灵脉是零和博弈
- **代际传承**: 老弟子坐化，新弟子成长，宗门精神延续

## 玩家动机（SDT + Bartle）

### 自决理论
| 需求 | 如何满足 |
|------|----------|
| **自主性** | 门规制定、弟子培养方向、宗门外教策略全由玩家选择 |
| **胜任感** | 从练气到飞升的成长曲线清晰，每一次突破都是成就感 |
| **相关性** | 弟子关系网、宗门故事线、与NPC宗门的恩怨情仇 |

### Bartle 分类
- **成就型 (40%)**: 宗门排名、境界突破、稀有丹药炼制 — 核心驱动力
- **探索型 (25%)**: 未知秘境、失传功法、隐藏弟子天赋
- **社交型 (20%)**: 弟子关系网、宗门外交
- **杀手型 (15%)**: 宗门战争、灵脉争夺

## 核心循环

### 30秒循环
查看左上角通知 → 指派弟子处理突发事件的单步操作

### 5-15分钟循环（单次游玩）
1. 查看各弟子修炼进度
2. 分配/调整弟子任务（修炼/炼丹/外出/镇守）
3. 检查资源和建造队列
4. 推进时间（1-3个月）
5. 处理触发的随机事件
6. 重复 1-5

### 30-120分钟循环（单次游玩会话）
一次闭关修炼期（10-30年游戏时间）→ 看到宗门明显变化：
- 一个弟子从入门到筑基
- 一座新设施建成
- 一次外交事件完成
- 宗门等级提升

### 长期循环
宗门代际传承 → 老一代弟子坐化 → 新一代弟子成长为长老 → 宗门从九品升至一品

## MVP 定义

### 必须（MVP v0.1）
- 基础宗门管理界面（设施列表、弟子列表）
- 弟子系统（属性、灵根、修炼进度、基础任务分配）
- 时间系统（推进月份、年度结算）
- 3个设施（修炼室、丹药堂、灵脉）
- 资源系统（灵石 + 基础灵草）
- 境界突破（练气→筑基→金丹→元婴）
- 5种随机事件

### 应该（v0.2-v0.3）
- 炼丹系统
- 功法体系
- 弟子关系网
- 世界地图 + NPC宗门
- 外交系统

### 可以（v0.4+）
- 炼器系统
- 阵法系统
- 灵兽系统
- 宗门战争
- 老祖飞升

### 不做
- 玩家操控单个角色战斗
- 开放世界探索
- 多人联网
- 付费抽卡

## 风险与未决问题

| 风险 | 严重度 | 缓解措施 |
|------|--------|----------|
| 弟子太多导致管理繁琐 | 中 | 批量操作、自动化任务模板、长老代理 |
| 后期重复性高 | 中 | 事件系统 + 代际变化保持新鲜感 |
| 世界动态平衡难 | 高 | 快速原型验证经济模型，大量模拟测试 |
| Godot 4 生态不如 Unity | 低 | 项目复杂度不需要第三方插件 |
