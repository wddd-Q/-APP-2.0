# 掌门总览 UI 方案

## 目标

把默认首页从“功能清单”改成“经营仪表盘”，让玩家一进入游戏就知道：

- 宗门现在状况如何。
- 本月最该处理什么。
- 哪些系统有红点或风险。
- 下一步如何推动经营和故事。

## 信息架构

### 顶部栏

保留全局状态：

- 日期。
- 灵石。
- 声望。
- 弟子数量。
- 推进一月。
- 短通知。

顶部栏不再堆放大量功能按钮。

### 左侧导航

统一放主要入口：

- 掌门总览。
- 弟子名册。
- 天下地图。
- 宗门纪事。
- 天下榜。
- 宗门全景。
- 秘境探索。
- 炼丹炼器。
- 仓库。
- 修真史记。
- 存档。

带红点的系统在按钮文字后显示提示。

### 主区域：掌门总览

第一版分为五块：

1. **宗门态势**
   - 宗门名。
   - 品级。
   - 天下排名。
   - 与上一名差距。

2. **本月账目**
   - 灵石库存。
   - 预计月结。
   - 资源摘要。

3. **掌门初任/阶段目标**
   - 当前目标进度。
   - 下一步。
   - 完成状态。

4. **待处理**
   - 待处理纪事数量。
   - 当前风险。
   - 点击进入宗门纪事。

5. **核心弟子**
   - 境界最高或最需要关注的弟子。
   - 当前任务。
   - 修为进度。
   - 可突破提示。

## 第一版实现边界

本轮先做：

- 左侧导航雏形。
- 顶部栏瘦身。
- 宗门总览内增加经营仪表盘卡片。
- 保留原有设施、弟子、资源列表作为下半区。

暂不做：

- 完整响应式重排。
- 全新图标系统。
- 堂口 UI。
- 弟子位置显示。

## 视觉方向

- 背景：深墨色或深棕黑。
- 强调色：金色、玉色、朱红红点。
- 卡片：低圆角、轻边框、深色底。
- 信息表达：优先用短句、数字、进度条、状态标签。

## 交互规则

- 进入游戏默认显示掌门总览。
- 有未读宗门纪事时，侧边栏显示红点。
- 点击侧边栏按钮打开对应页面或弹层。
- 目标完成后总览即时刷新。
- 信息卡只显示摘要，详情通过按钮进入对应系统。
