# ADR-001: 存档格式选择

## 状态
已采纳 (2026-05-27)

## 背景
需要选择游戏存档的序列化格式。候选方案：JSON、Godot Resource (.tres)、二进制 (var_to_bytes)。

## 决策
**选择 JSON 格式。**

## 评估标准

| 方案 | 可读性 | 跨版本兼容 | 体积 | 复杂度 |
|------|--------|-----------|------|--------|
| JSON | ★★★★★ | ★★★★☆ | ★★★☆☆ | ★★★★★ |
| Resource (.tres) | ★★★★☆ | ★★☆☆☆ | ★★★★☆ | ★★★☆☆ |
| 二进制 | ★☆☆☆☆ | ★★☆☆☆ | ★★★★★ | ★★☆☆☆ |

## 理由

1. **可调试性**: 玩家可以直接打开存档文件查看/报告问题
2. **版本迁移**: JSON 结构可以通过迁移脚本处理字段变更
3. **跨平台一致**: 纯文本，无端序问题
4. **够用的性能**: 预期存档 < 2MB，JSON 解析毫秒级

## 后果

- 正向: 易于调试、手动修复存档、mod支持
- 负向: 文件体积比二进制大 3-5x，但对于模拟经营游戏可接受
- 风险: 需要在版本升级时维护迁移逻辑

## 替代方案

**Godot Resource (.tres)**: 放弃了。因为 Resource 序列化格式不是公开稳定的API，Godot 版本升级可能导致存档不兼容。

**二进制**: 放弃了。性能优势在本游戏场景中无意义（不是即时存档/读档密集的游戏），但调试成本显著增加。
